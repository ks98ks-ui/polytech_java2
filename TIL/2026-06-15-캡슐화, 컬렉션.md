# 2026-06-15 캡슐화, 컬렉션

## 오늘 배운 내용

- 실수를 방지하기 위해 캡슐화 사용
- getter, setter 로 읽기, 쓰기 구현
- 필드는 private 메소드는 public
- List는 순서대로 쌓여있는 구조, Map은 키와 값의 쌍으로 저장, Set은 순서가 없는 집합(중복불가)
- List는 추가하면 알아서 저장공간이 커지지만 배열은 추가하거나 뺄 때 새로 만들어야 한다.

## 실습 코드

```java
public class ListExam {
    public static void main(String[] args) {
        Hero hero1 = new Hero();
        Hero hero2 = new Hero();
        Hero hero3 = new Hero();

        ArrayList<Hero> heroes = new ArrayList<>();
        heroes.add(hero1);
        heroes.add(hero2);
        heroes.add(hero3);

        System.out.println(heroes.size());
        System.out.println(hero1.getName());
        System.out.println(heroes.get(0).getName());

        ArrayList<Integer> numbers = new ArrayList<>();
        numbers.add(1);
        numbers.add(1);
        numbers.add(1);


        System.out.println(numbers);
        System.out.println(numbers.contains(1));
        HashSet<Integer> numberSet = new HashSet<>();
        numberSet.add(1);
        numberSet.add(1);
        numberSet.add(1);
        System.out.println(numberSet);
        System.out.println(numberSet.contains(2));

    }
}
```

## 어려웠던 점

- Assertion 종류를 잘 알지 못해 테스트코드를 짤 때 어떤 것을 써야 하는지 잘 몰랐던것
- 테스트 코드 짜는게 아직 익숙치 않은것

## 해결 방법

- Assertion종류 검색
- Given When Then 으로 나눠서 차례대로 코드작성

## 내일 더 공부할 것

- Assertion종류를 제대로 알고 적절한 곳에 사용
- 저번주에 배웠던 것들 복습
